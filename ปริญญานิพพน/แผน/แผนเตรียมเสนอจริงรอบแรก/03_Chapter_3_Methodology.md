# บทที่ 3: วิธีดำเนินการวิจัย (Methodology)

การวิจัยนี้ใช้ระเบียบวิธีวิจัยเชิงปริมาณ (Quantitative Research) และการสร้างแบบจำลองทางวิศวกรรม (Engineering Modeling) โดยมีขั้นตอนการดำเนินงานที่สอดคล้องกับกรอบแนวคิด V4 ซึ่งจัดโครงสร้างเป็นกระบวนการพัฒนาอย่างต่อเนื่องแบบ **PDCA 3 วงจร (3-Phase PDCA Cycle)** ดังนี้

## 3.1 วงจร PDCA ที่ 1: การล้างไพ่ข้อมูลและการสร้างเส้นฐานความจริง (Work Study & OEE Baseline)

เป้าหมายของ PDCA 1 คือการได้มาซึ่งข้อมูลความจริง (Delivery / Baseline) โดยมีขั้นตอนดังนี้:

1.  **Plan:** ทบทวนเวลามาตรฐาน (SCT) เดิม และออกแบบการศึกษาเวลา (Time Study) โดยใช้กล้องวงจรปิดเพื่อลดผลกระทบ Hawthorne Effect พร้อมทั้งเตรียมกระบวนการขอความยินยอม (Ethics) จากพนักงาน
2.  **Do:** ดำเนินการจับเวลาการทำงานไม่น้อยกว่า 30 วงจร (Cycles) โดยแยกเวลาที่เครื่องจักรทำงานอัตโนมัติ (Auto Machining Time) ออกจากเวลาที่คนทำงาน (Manual Time) อย่างชัดเจน
3.  **Check:** 
    *   ประเมินอัตราความเร็วและบวกเวลาเผื่อ (Allowance) เพื่อสร้าง SCT ใหม่
    *   คำนวณประเมิน OEE แบบปรับตามความต้องการ (Demand-adjusted OEE ตามมาตรฐาน ISO 22400-2)
4.  **Act:** ส่งมอบชุดข้อมูล SCT และ ICT ที่ถูกต้อง พร้อมทั้งรายงาน OEE Baseline ที่สะท้อนโลกความเป็นจริงให้แก่ผู้บริหาร (เพื่อตอบสมมติฐาน $H_1$)

## 3.2 วงจร PDCA ที่ 2: การวิเคราะห์ความน่าเชื่อถือและจุดคุ้มทุน (Reliability Analysis & Regrind Policy)

เป้าหมายของ PDCA 2 คือการวิเคราะห์ต้นทุนเชิงลึกและการบำรุงรักษาเชิงป้องกัน (Cost & Availability) ของมีดด้านเกลียว (Thread Tool):

1.  **Plan:** ออกแบบการเก็บข้อมูลอายุมีดกลึงตามมาตรฐาน ISO 3685 (พิจารณาจากรอยสึก Flank Wear หรือมิติหลุด Tolerance) โดยใช้วิธี Observational Piggyback (เกาะติดการผลิตจริง ไม่รันให้พังคามือเพื่อรักษาเครื่องจักร)
2.  **Do:** 
    *   บันทึกข้อมูลอายุมีดกลึง (Cycles) พร้อมจำแนกโหมดการเสื่อมสภาพเป็นโหมดสึกหรอ (Wear-out) และโหมดแตกหัก (Catastrophic)
    *   บันทึกประวัติการลับมีดซ้ำ (Regrind History) และข้อจำกัดทางเรขาคณิตของมีด
    *   ประเมินสถานะปัจจุบันของเวลาตั้งเครื่อง (Baseline SMED)
3.  **Check:** 
    *   **Stratified Weibull & Competing Risks:** ประเมินพารามิเตอร์ $\beta$ และ $\eta$ ของแต่ละโหมดด้วยวิธี Median Rank Regression (MRR) พร้อมการปรับแก้ Bias (ตอบสมมติฐาน $H_2$)
    *   **Cost Optimization ($t_p^*$):** ใช้ Python `scipy.optimize` หาค่า $t_p^*$ ที่ทำให้ฟังก์ชันต้นทุนรวม (Renewal-Reward Cost Rate) ต่ำที่สุด (ตอบสมมติฐาน $H_3$)
    *   **Regrind Limit ($N_{max}$):** ผสานข้อจำกัดทางเรขาคณิตเข้ากับจุดคุ้มทุนเชิงเศรษฐศาสตร์ เพื่อหากำหนดจำนวนครั้งลับมีดสูงสุด (ตอบสมมติฐาน $H_4$)
4.  **Act:** ร่างมาตรฐาน Preventive Maintenance (PM) และนโยบายการลับมีด พร้อมนำหลักการ SMED มาวิเคราะห์เพื่อแปลง Internal Setup เป็น External Setup ลดเวลาเปลี่ยนมีด

## 3.3 วงจร PDCA ที่ 3: การควบคุมคุณภาพและการพิสูจน์ผล (Quality Control & Confirmation Run)

เป้าหมายของ PDCA 3 คือการรับรองคุณภาพชิ้นงาน และพิสูจน์ผลลัพธ์การประหยัด (Quality & Validation):

1.  **Plan:** วางแผนการวัดมิติวิกฤตของชิ้นงาน (Critical Dimension) ตลอดช่วงอายุการใช้งานของมีด และทำการทดสอบระบบการวัด (Gage R&R) ทั้งของมิติชิ้นงานและรอยสึก (VB)
2.  **Do:** ทำการสุ่มตัวอย่างชิ้นงานเป็นกลุ่มย่อย (Subgroup) ตามช่วงการใช้งานของมีด (เช่น ทุกๆ 500 ชิ้น) และบันทึกผลการวัด
3.  **Check:**
    *   **SPC Analysis:** นำข้อมูลมาสร้างแผนภูมิควบคุม Regression-adjusted Control Chart / CUSUM เพื่อตรวจสอบว่าการยืดอายุมีดไปยัง $t_p^*$ กระบวนการยังคงอยู่ในสภาวะ Statistical Control หรือไม่
    *   **Optimal Inspection:** วิเคราะห์รอบการใช้ Plug Gauge สุ่มตรวจ เพื่อรักษาสมดุลระหว่างต้นทุนการตรวจกับมูลค่าการเกิด Scrap
4.  **Act:**
    *   **Confirmation Run (ทดสอบวิ่งจริง):** ดำเนินการนำค่า $t_p^*$ และ $N_{max}$ ไปใช้ผลิตจริงเพื่อประเมินผลต่าง (Delta) ระหว่างต้นทุนต่อชิ้นที่ทำนายไว้ (Predicted CPP) กับต้นทุนที่เกิดขึ้นจริง (Actual CPP) (ตอบสมมติฐาน $H_5$)
    *   **Dashboard PoC:** สร้างต้นแบบหน้าปัดแสดงผลเพื่อช่วยผู้ปฏิบัติงานในการติดตามอายุมีดและการดำเนินการตามนโยบาย PM ใหม่

## 3.4 การประเมินคอขวดนอกเครื่องจักร (System Constraint Evaluation)

นอกเหนือจาก 3 PDCA หลัก งานวิจัยยังประเมินผลกระทบเชิงระบบ (System Impact) จากความสูญเปล่าที่เรียกว่า "ไม่มีลังจากแผนกประกอบ" ซึ่งเป็น Downtime สูงสุด
ตามหลักการของ Lean Manufacturing และ ISO 22400-2 ปรากฏการณ์นี้คือ **"การหยุดตามดีมานด์ (No-demand stop)"** การเปิดเครื่อง Standby รอไว้ก่อให้เกิดต้นทุนแฝง งานวิจัยจะแสดงให้เห็นค่าเสียโอกาสทางเศรษฐศาสตร์ (Machine-Hour Rate Opportunity Cost) จากการ Idling เพื่อเสนอแนะแก่ผู้บริหารให้ทราบถึงปัญหาคอขวดในกระบวนการถัดไป (Downstream Constraint) โดยไม่นับความสูญเสียนี้เป็นความล้มเหลวของเครื่อง CNC MACOD 1569 เอง
