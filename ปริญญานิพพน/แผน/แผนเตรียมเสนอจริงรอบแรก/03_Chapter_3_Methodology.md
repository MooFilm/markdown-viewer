# บทที่ 3: วิธีดำเนินการวิจัย (Methodology)

> **หมายเหตุการปรับทิศ (v2 — ปรับตามข้อมูลจริงหน้างาน):** บทนี้ปรับให้สอดคล้องกับ [Thesis_Framework_v2_TH.md](../Thesis_Framework_v2_TH.md) โดยมี 3 จุดเปลี่ยนหลัก: (1) **มีดปาดหน้า (Face Tool) เลิกใช้แล้ว** เนื่องจากเปลี่ยนดีไซน์เป็นแชมเฟอร์ → การวิเคราะห์มีดโฟกัสที่ **มีดด้านเกลียว (Thread Tool)** เป็นหลัก; (2) Downtime "ไม่มีลังจากประกอบ" (82%) จัดเป็น **การหยุดตามดีมานด์ (No-demand / กัน WIP)** ไม่ใช่ความสูญเสียที่ต้องกู้คืน; (3) การหาจุดเปลี่ยนมีดคุ้มสุดใช้ **วิธีเชิงตัวเลข (numerical optimization)** ด้วย Python แทนการหาอนุพันธ์ด้วยมือ

การวิจัยนี้ใช้ระเบียบวิธีวิจัยเชิงปริมาณ (Quantitative Research) และการสร้างแบบจำลองทางวิศวกรรม (Engineering Modeling) โดยมีขั้นตอนการดำเนินงานที่สอดคล้องกับวงจร PDCA (Plan-Do-Check-Act) และแบ่งออกเป็น 5 ระยะ (Phases) ดังนี้

## 3.1 ระยะที่ 1: การล้างไพ่ข้อมูลและการสร้างเส้นฐานความจริง (Data Rectification & Truth Baseline)

เนื่องจากระบบการประเมินประสิทธิภาพเดิมมีข้อบกพร่องเรื่องเวลามาตรฐาน (Standard Cycle Time: SCT) ผู้วิจัยจึงต้องทำการศึกษาเวลาใหม่ด้วยวิธีย่อยองค์ประกอบงาน (Micro-motion study):
1.  **การกำหนดขนาดตัวอย่าง (Sample Size Determination):** คำนวณจำนวนรอบการจับเวลาที่เหมาะสม เพื่อให้ข้อมูลมีความน่าเชื่อถือทางสถิติที่ระดับความเชื่อมั่น 95% และความคลาดเคลื่อนยอมรับได้ $\pm 5\%$ โดยใช้สมการสถิติสำหรับการศึกษาเวลา:
    $$N' = \left( \frac{40 \sqrt{N \sum X^2 - (\sum X)^2}}{\sum X} \right)^2$$
    เมื่อ $N'$ คือจำนวนตัวอย่างที่ต้องการ, $N$ คือจำนวนตัวอย่างเบื้องต้น (เช่น 30 รอบ), และ $X$ คือเวลาที่จับได้ในแต่ละรอบ
2.  **การแยกองค์ประกอบงาน (Work Element Breakdown):** แยกแยะเวลาที่เครื่องจักรทำงานอัตโนมัติ (Auto Machining Time) ออกจากเวลาที่คนทำงาน (Manual Time เช่น การใส่/ถอดชิ้นงาน การเป่าลมทำความสะอาด)
3.  **การประเมินอัตราความเร็วและการให้เวลาเผื่อ (Rating & Allowances):** ประเมินอัตราความเร็วของพนักงานด้วยวิธี Westinghouse System และบวกเวลาเผื่อความเหนื่อยล้า (Fatigue Allowance)
4.  **การคำนวณ OEE ใหม่ (OEE Recalculation):** นำ SCT ที่ถูกต้องมาคำนวณ OEE ย้อนหลัง เพื่อหาค่า Performance Rate ที่แท้จริง และแสดงให้เห็นถึงช่องว่างการพัฒนา (Improvement Gap)

## 3.2 ระยะที่ 2: การวิเคราะห์ความน่าเชื่อถือของมีดกลึง (Reliability Analysis)

เพื่อยกเลิกการใช้ "ค่าเฉลี่ยแบบเหมารวม" ผู้วิจัยจะทำการสร้างสมการการเสื่อมสภาพของมีดกลึง โดย **โฟกัสที่มีดด้านเกลียว (Thread Tool)** ซึ่งเป็นมีดที่ยังใช้งานกับ HC15-25 ในปัจจุบัน (มีดปาดหน้าเลิกใช้แล้วจากการเปลี่ยนดีไซน์เป็นแชมเฟอร์):
1.  **การเก็บข้อมูลอายุตามจริง (Run-to-failure Data):** รวมข้อมูลอายุมีดเดิม 9 ค่าจาก [เปลี่ยนใบมีดเครื่อง1569.csv](../../รายงาน/ข้อมูลประกอบ/เปลี่ยนใบมีดเครื่อง1569.csv) (เฉลี่ย ~13,991 ชิ้น, CV ~24%) เข้ากับการทดลอง **เบิกมีดเกลียวใหม่ 5 ตัวเดินจนสิ้นสภาพ (run-to-failure)** ตามคำแนะนำอาจารย์ที่ปรึกษา และจัดทำ Check sheet ([02_Tool_Life_RunToFailure_Template.csv](../../รายงาน/ข้อมูลประกอบ/templates/02_Tool_Life_RunToFailure_Template.csv)) บันทึกอายุ (Cycles), **ชนิด (มีดใหม่ vs มีดลับมา) และจำนวนครั้งที่ลับ** เพื่อให้แยกพฤติกรรมการเสื่อมตามจำนวนรอบลับได้
2.  **การพล็อตกราฟและหาพารามิเตอร์ (Parameter Estimation):** นำข้อมูลอายุมีด $t_1, t_2, ..., t_n$ มาทำการวิเคราะห์แบบ Maximum Likelihood Estimation (MLE) หรือ Rank Regression เพื่อหาค่าของการแจกแจงแบบไวบูลล์:
    *   $\beta$ (Shape parameter): ประเมินรูปแบบการเสีย
    *   $\eta$ (Scale parameter): ประเมินอายุขัยที่ 63.2% 
3.  **การทดสอบความเหมาะสมของการแจกแจง (Goodness-of-Fit Test):** แทนที่จะใช้เพียง R-squared งานวิจัยนี้จะใช้การทดสอบ **Anderson-Darling (AD) Test** เพื่อเปรียบเทียบการแจกแจงแบบ Weibull, Lognormal และ Exponential ว่าแบบใดมีความสอดคล้องกับข้อมูลอายุมีดจริงมากที่สุด โดยพิจารณาจากค่า AD statistic ที่ต่ำที่สุดและ p-value > 0.05

## 3.3 ระยะที่ 3: การสร้างแบบจำลองสมการหาจุดคุ้มทุน (Mathematical Cost Optimization)

เมื่อทราบพฤติกรรมการเสื่อมสภาพ ผู้วิจัยจะนำมาสร้างแบบจำลองหาค่าใช้จ่ายรวมต่ำสุดในการบำรุงรักษาเชิงป้องกัน (Optimal PM Interval)
1.  **รวบรวมต้นทุนฐาน (Cost Baseline Collection):**
    *   $C_p$ (ต้นทุนเปลี่ยนมีดตามแผน) = ค่ามีดกลึง 1 เม็ด + (เวลาในการทำ SMED เปลี่ยนมีด $\times$ อัตราค่าเสียโอกาส Machine-Hour Rate)
    *   $C_f$ (ต้นทุนมีดแตกและเกิดของเสีย) = ค่ามีดกลึง 1 เม็ด + (เวลา Downtime ซ่อมแซมเครื่อง $\times$ Machine-Hour Rate) + ค่าของเสียชิ้นงาน HC15-25 (Scrap Cost)
2.  **การสร้างฟังก์ชันและการหาค่าเหมาะที่สุดเชิงตัวเลข (Modeling & Numerical Optimization):** นำ $C_p$ และ $C_f$ ไปแทนในสมการ Total Cost Per Part ($CPP(t_p)$) ตามแบบจำลอง Age-Replacement จากนั้นหาจุดต่ำสุดด้วย **วิธีเชิงตัวเลข (grid search / `scipy.optimize`)** ผ่านสคริปต์ Python ใน [รายงาน/scripts/](../../รายงาน/scripts/) เพื่อหาจำนวนชิ้นงาน $t_p^*$ ที่ทำให้ $CPP(t_p)$ ต่ำสุด — วิธีนี้ไม่ต้องหาอนุพันธ์ด้วยมือ ลดความเสี่ยงด้านการคำนวณ และผู้วิจัยทำหน้าที่ตรวจสอบผล (review) ด้วยข้อมูลจริง
3.  **นโยบายการลับมีดซ้ำ (Regrind Policy):** วิเคราะห์การลดลงของอายุมีดต่อจำนวนครั้งที่ลับ เพื่อกำหนด **จำนวนครั้งลับสูงสุดที่ยังคุ้ม** (เทียบต้นทุนลับ vs ซื้อใหม่) ก่อนตัดสินใจ scrap มีด
4.  **การลดเวลาตั้งเครื่อง (SMED Application):** เพื่อลดค่า $C_p$ ลงให้เหลือน้อยที่สุด ผู้วิจัยจะประยุกต์ใช้หลักการ SMED ในจังหวะเปลี่ยนมีดกลึง โดยแปลงงานภายใน (Internal Setup) เป็นงานภายนอก (External Setup) ให้มากที่สุด

## 3.4 ระยะที่ 3.5: การตรวจสอบเสถียรภาพคุณภาพด้วย SPC (Quality Stability Verification)

เพื่อเป็นการรับรองคุณภาพ (Quality Assurance) ว่าการยืดอายุหรือปรับเปลี่ยนจุดเปลี่ยนมีดกลึงตามสมการ $t_p^*$ จะไม่ส่งผลเสียต่อคุณภาพมิติของชิ้นงาน:
1.  **การเก็บข้อมูลกลุ่มตัวอย่าง (Subgroup Sampling):** สุ่มเก็บตัวอย่างชิ้นงาน HC15-25 จำนวน 5 ชิ้นต่อกลุ่ม (Subgroup size $n=5$) เป็นระยะๆ ตามช่วงการใช้งานของมีดกลึง ตั้งแต่ชิ้นแรกจนถึงจำนวนชิ้นงานที่ $t_p^*$
2.  **การสร้างแผนภูมิควบคุม (Control Chart Construction):** นำค่ามิติวิกฤต (Critical Dimension) มาพล็อตกราฟ $\bar{X}$-R Chart โดยคำนวณเส้นขอบเขตควบคุม (Control Limits) จากข้อมูลที่เก็บได้
3.  **การวิเคราะห์เสถียรภาพ (Stability Analysis):** วิเคราะห์ตามกฎ Western Electric Rules เพื่อตรวจสอบว่ากระบวนการอยู่ในสถาวะควบคุมเชิงสถิติ (In-control) หรือไม่ หากกราฟแสดงแนวโน้ม (Trend) การเลื่อนตัวของค่าเฉลี่ยที่ชัดเจนก่อนถึงจุด $t_p^*$ อาจต้องพิจารณาปรับค่า $t_p^*$ ใหม่เพื่อป้องกันการเกิดของเสีย

## 3.5 ระยะที่ 4: การจำแนกประเภทการหยุดเครื่องและข้อเสนอแนะเชิงระบบ (Stop Classification & System Recommendations)

> **ข้อปรับสำคัญ (v2):** Downtime อันดับ 1 "ไม่มีลังจากประกอบ" (82%) เกิดจากแผนกประกอบสลับไปประกอบโมเดลอื่น หากเครื่อง CNC เดินต่อจะกลายเป็น WIP/Overproduction ดังนั้นการหยุดนี้คือ **"การหยุดตามดีมานด์ (No-demand stop)" ซึ่งถูกต้องตามหลัก Lean** และ **ไม่นับเป็น Availability loss** ในการคำนวณ OEE (ดูนิยาม demand-adjusted ในข้อ 4.1 ของ Framework v2) จึง **ไม่ใช่เป้าหมายการกู้คืน** ของปริญญานิพนธ์ แต่ทำเป็นข้อเสนอแนะเชิงระบบเท่านั้น

1.  **การจำแนกประเภทการหยุด (Stop Classification):** แยกเวลาหยุดออกเป็น (ก) Planned/No-demand (ไม่มีลัง, พักตามระเบียบ), (ข) Unplanned loss (มีดพัง, แขนกลชน, ซ่อม) เพื่อให้ OEE สะท้อนเฉพาะความสูญเสียที่ควบคุมได้จริง
2.  **การประเมินต้นทุน Idling เชิงเสริม (Supplementary Idling Cost):** ประเมินค่าพลังงาน (kW × ชม. × บาท/kWh) ของช่วงเครื่องเปิดสแตนด์บายรอลังเป็นข้อมูลประกอบ (ไม่ใช่ตัวชี้วัดหลัก) เพื่อชี้ว่าหากเปิดเครื่องทิ้งไว้โดยไม่จำเป็นยังมีค่าใช้จ่ายแฝง
3.  **ข้อเสนอแนะเชิงกลยุทธ์ (Strategic Recommendations):** เสนอผู้บริหารพิจารณาการปิดเครื่อง/standby อัตโนมัติช่วงไม่มีดีมานด์ หรือใช้กำลังการผลิตส่วนเกินไปงานอื่นด้วย SMED — เป็นข้อเสนอแนะนอกขอบเขตหลัก (เครื่องเดียว)
